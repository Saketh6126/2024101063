\section{White-Box Test Cases}
\label{sec:whitebox}

\subsection{Requirement coverage}
This section addresses the requirement:\\
\textit{``Design white-box test cases based on the structure of the code. Your tests must cover: (i) All branches, (ii) Key variable states that could affect program behavior, (iii) Relevant edge cases. In the report, explain in simple words why each test case is needed, along with all the errors or logical issues found by the test.''}

\subsection{Where the tests live (as per directory requirement)}
Per the required directory structure, the white-box test suite is located at:
\begin{itemize}[leftmargin=*]
  \item \texttt{whitebox/tests}
\end{itemize}

\subsection{How tests were run}
Because the importable package lives under \texttt{whitebox/code}, the test runner is executed from \texttt{whitebox/} and \texttt{whitebox/tests/conftest.py} adds \texttt{whitebox/code} to the import path:

\begin{verbatim}
cd whitebox
python -m pytest -q tests
\end{verbatim}
